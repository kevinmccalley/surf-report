# Surf Forecasting — Domain Knowledge for Claude

This document gives Claude deep context about wave physics, forecast interpretation, and
Groundswell's specific data model so that AI features in this codebase produce accurate,
surfing-literate output.

**Update this file whenever:** a new data field is added, scoring thresholds change, a new
API source is wired up, or the display/unit conventions change.

---

## 1. Groundswell's Data Pipeline

**Source:** Open-Meteo exclusively (marine API + weather API). No Stormglass or NOAA direct calls.

**Marine models used:**
- Primary: `best_match` (NEMO; falls back to ECMWF WAM)
- Secondary fallback: `ecmwf_wam` (14-day range)
- Day-15 gap-fill: `ncep_gfswave025` (GFS)

**Units from API:** wave heights in **meters**, periods in **seconds**, directions in **degrees (0–359°)**, wind speed in **km/h**, temperature in **°C**.

**Unit displayed to users:** feet (converted via `metersToFeet()` in `app/lib/utils.ts`). No Hawaiian scale; no face-height scaling. Raw model meters → feet.

## 2. Wave Data Fields

| App field | API source field | Meaning |
|---|---|---|
| `waveHeight` | `wave_height` | Total significant wave height (Hs) — combined sea + swell |
| `wavePeriod` | `wave_period` | Dominant wave period (seconds) |
| `primarySwell.height` | `swell_wave_height` | Primary swell component Hs |
| `primarySwell.period` | `swell_wave_period` | Primary swell period |
| `primarySwell.direction` | `swell_wave_direction` | Direction swell is traveling **from** (degrees) |
| Secondary swell (swell_2) | `swell_wave_height_2/period_2/direction_2` | Secondary swell train |
| `secondarySwell` (wind sea) | `wind_wave_height/period/direction` | Locally generated chop |
| `wind.speed` | `wind_speed_10m` | 10m wind speed (km/h) |
| `wind.gust` | `wind_gusts_10m` | Gust speed (km/h) |
| `wind.direction` | `wind_direction_10m` | Direction wind is blowing **from** (degrees) |
| `waterTemp` | `sea_surface_temperature` | SST in °C |
| `airTemp` | `temperature_2m` | Air temp at 2m (°C) |

**Swell direction convention:** degrees indicate where the swell is coming **from**. 0° = North, 90° = East, 180° = South, 270° = West. A swell at 270° is a west swell traveling eastward.

## 3. Surf Rating System (`app/lib/surf-rating.ts`)

Conditions are scored on three independent components summed to a max of 10:

### Height score (0–4 pts)
| Range (meters) | Score |
|---|---|
| < 0.3 m (< ~1 ft) | 0.5 |
| 0.3–0.6 m (~1–2 ft) | 1.5 |
| 0.6–0.9 m (~2–3 ft) | 2.5 |
| 0.9–1.5 m (~3–5 ft) | 3.2 |
| 1.5–2.0 m (~5–6.5 ft) | 3.7 |
| 2.0–3.0 m (~6.5–10 ft) | **4.0 peak** |
| 3.0–4.5 m (~10–15 ft) | 3.8 |
| > 4.5 m (> 15 ft) | 3.2 (decreasing returns) |

### Period score (0–3 pts)
| Period (seconds) | Score | What it means |
|---|---|---|
| < 6 s | 0 | Wind chop only — no real swell |
| 6–8 s | 0.6 | Short-period wind swell |
| 8–10 s | 1.2 | Building swell, still choppy |
| 10–12 s | 1.7 | Moderate groundswell |
| 12–15 s | 2.2 | Good groundswell |
| 15–18 s | 2.7 | Long-period groundswell |
| ≥ 18 s | 3.0 | Deep-water energy, long paddle-out |

### Wind score (0–3 pts)
| Wind speed (km/h) | Score | Surface condition |
|---|---|---|
| < 8 | 3.0 | Glassy |
| 8–15 | 2.4 | Light texture |
| 15–22 | 1.6 | Textured / some chop |
| 22–30 | 0.8 | Choppy |
| 30–40 | 0.3 | Very choppy |
| > 40 | 0 | Blown out |

### Rating labels (composite score → label)
| Score range | Label | Color |
|---|---|---|
| 0.0 | FLAT | Gray |
| 1.5–3.5 | POOR | Red |
| 3.5–5.0 | POOR TO FAIR | Orange |
| 5.0–6.5 | FAIR | Yellow |
| 6.5–7.8 | FAIR TO GOOD | Lime |
| 7.8–9.0 | GOOD | Green |
| 9.0+ | VERY GOOD / EPIC | Cyan / Purple |

## 4. Spot Calibration (`app/lib/spot-calibration.ts`)

The generic score is post-processed against 27 curated world-class breaks. Three caps can be applied:

1. **Minimum swell height cap** — if `swell_wave_height < minSwellM`, rating capped at POOR regardless of period or wind. Example: Pipeline requires ≥ 1.5 m swell.

2. **Direction window cap** — each break has an optimal swell direction range `[dirMin, dirMax]`.
   - 0–25° outside window → capped at FAIR
   - > 25° outside window → capped at POOR TO FAIR
   - Wraps through 0° for north-facing breaks (e.g., Thurso: 345–60°)

3. **Minimum period cap** — if period < `minPeriod`, rating drops one tier. Example: Pipeline needs ≥ 12 s.

Fuzzy matching uses a `matchRadiusKm` (typically 3 km) so nearby coordinates still trigger calibration.

## 5. Wave Physics Primer (for AI reasoning)

### Groundswell vs. wind swell
- **Groundswell** — generated by distant storms; long fetch; period typically > 12 s. Arrives as clean, organized lines. More energy per wave. Preferred for surfing.
- **Wind swell** — generated locally by regional winds; short period (< 10 s); choppy, disorganized. Less powerful but can still be surfable.

### Significant wave height (Hs)
The average of the top 1/3 of waves by height. About 1/3 of waves will be taller than Hs. "Rogue" waves can reach 2× Hs. Open-Meteo reports Hs in meters.

### Period and wave energy
Wave energy scales with period squared — a 16 s wave carries ~2.8× the energy of an 8 s wave at the same height. Long period + moderate height often surfs bigger than short period + similar height.

### Wave propagation speed
Long-period swells travel faster and arrive earlier than model predictions sometimes suggest. A 20 s swell moves at ~30 knots; a 10 s swell at ~15 knots. Use period to judge when a swell will peak.

### Direction and break orientation
Every break has a facing direction. A swell from 270° (west) hits a west-facing beach directly (optimal) but would be blocked from an east-facing beach. Oblique angles can produce better walls than direct angles depending on the break's geography.

### Wind direction relative to the break
- **Offshore** — wind blowing from land to sea (into the face of breaking waves). Grooms the wave face, holds lips up. Ideal conditions.
- **Onshore** — wind blowing from sea to land. Pushes waves over early, creates chop. Degrades quality.
- **Cross-shore** — side wind. Neutral to slightly negative depending on strength.

Whether a given wind direction is offshore depends entirely on the break's orientation. A north wind is offshore at a south-facing beach and onshore at a north-facing beach.

### Tides
Groundswell does not currently fetch tide data. Tide affects wave quality significantly — many breaks are tide-sensitive (e.g., reef breaks work best at mid-tide; some beach breaks prefer low tide). This is a known limitation.

### Wave period vs. interval
"Period" in Open-Meteo is the dominant period across the full spectrum. "Interval" (used colloquially) is the time between sets. These are related but not identical — a 15 s period swell may produce set intervals of 45–60 s due to wave grouping.

## 6. Swell Alert Logic (`app/api/cron/swell-alert-check/route.ts`)

- Threshold stored per saved location as `alertThreshold` in meters.
- Alert fires when `swell_wave_height >= alertThreshold`.
- 20-hour cooldown between alerts per location to prevent spam.
- Email sent via Loops.so event `swell_threshold_alert` with: `spotName`, `waveHeightFt`, `waveHeightM`, `thresholdFt`, `thresholdM`.

## 8. Forecast Interpretation Guidelines

When Claude is answering a question about surf conditions, it should:

1. **Lead with the rating label and height** — e.g., "FAIR TO GOOD, 3–4 ft". Users orient on this first.
2. **Cite period as the quality signal** — height gets clicks, period determines feel. A 3 ft wave at 16 s surfs much better than 3 ft at 7 s.
3. **Explain wind direction relative to the specific break** — don't just say "light winds"; say "light offshore wind" or "light onshore" based on break orientation.
4. **Flag calibration caps** — if a spot is being downgraded due to swell direction or minimum height, say so explicitly so the user understands why a "decent looking" swell is rated lower.
5. **Acknowledge tide blindspot** — if a user asks about a tide-sensitive break, note that tide data is not currently included in the forecast.
6. **Convert units for the user's preference** — default is feet; use meters if the user asks.
7. **Be honest about forecast window** — beyond 5–7 days, model accuracy degrades significantly. Day 10–15 is directional only.

## 9. Metrics for AI Evaluation

When assessing whether a session is worth paddling out, Claude should weigh:

| Factor | Weight | Notes |
|---|---|---|
| Swell period | High | Below 8 s → likely not worth it unless you're learning |
| Swell height | High | Scaled to the surfer's level |
| Wind speed/type | High | Offshore < 15 km/h = prime; onshore > 20 km/h = skip |
| Swell direction | Medium | Must align with break orientation |
| Crowd (not in data) | Medium | Not modeled; user must factor in |
| Tide (not in data) | Medium | Not modeled; user must factor in |
| Water temp | Low | Affects gear choice, not wave quality |
